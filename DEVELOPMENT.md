# openStudy Development Guide

## Project Overview

openStudy is a terminal UI (TUI) application inspired by OpenCode, designed to help with software engineering tasks directly from the command line.

## Current Setup

- **Framework**: Ink (React for the terminal)
- **Language**: TypeScript
- **Package Manager**: npm
- **Build Tool**: TypeScript compiler + tsx for development

## Project Structure

```
openstudy/
├── src/
│   ├── components/        # Reusable Ink components (empty - to be implemented)
│   ├── hooks/            # Custom React hooks (empty - to be implemented)
│   ├── types/            # TypeScript type definitions (empty - to be implemented)
│   ├── utils/            # Utility functions (empty - to be implemented)
│   └── index.tsx         # Application entry point
├── dist/                 # Compiled output (generated by build)
├── package.json          # Project metadata and dependencies
├── tsconfig.json         # TypeScript configuration
├── .gitignore           # Git ignore rules
└── README.md            # Project readme
```

## Available Commands

```bash
# Development mode with hot reload
npm run dev

# Build TypeScript to JavaScript
npm run build

# Run compiled application
npm start
```

## Development Tips

- Use `src/components/` for reusable UI components
- Use `src/hooks/` for custom React hooks to manage state and effects
- Use `src/types/` for TypeScript interfaces and types
- Use `src/utils/` for helper functions
- Always run `npm run dev` to test changes
- Use `npm run build` before committing to ensure no TypeScript errors

## Key Dependencies

- **ink**: React renderer for terminal
- **react**: UI library
- **typescript**: Type checking
- **tsx**: TypeScript executor for development

## Notes

- The application is configured to use ES modules (type: "module")
- tsx is used for development to run TypeScript directly without compilation
- tsc is used for building production code
